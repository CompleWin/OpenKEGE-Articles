# OpenKEGE Articles

Статьи по информатике для подготовки к ЕГЭ. Сайт на Astro, изначально просто
под чтение, но постепенно к нему прикрутились квизы и редактор Python,
код можно запускать прямо в браузере. Что ещё понадобится, то и допишу.

## Стек

- **Astro 6** - основной фреймворк
- **React 19** для интерактивных кусков (квизы, редактор)
- **MDX**, потому что в Markdown хочется иногда вкрячить JSX
- **Tailwind CSS 4** - стили
- **KaTeX** - формулы
- **Pyodide** - гоняет Python прямо в браузере
- **Supabase** считает лайки под статьями

## Запуск локально

```bash
# 1. Клонируешь
git clone https://github.com/CompleWin/OpenKEGE-Articles.git
cd OpenKEGE-Articles

# 2. Ставишь зависимости
npm install

# 3. Создаёшь .env (см. следующий раздел)
cp .env.example .env

# 4. Поехали
npm run dev
```

Сайт откроется на `http://localhost:4321`.

## Переменные окружения

В корне нужен `.env` с двумя переменными, иначе не работает кнопка лайков:

```env
PUBLIC_SUPABASE_URL=https://your-project.supabase.co
PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

Если ты пришёл просто писать статью и до лайков тебе дела нет, оставляй пустыми.
Сайт всё равно соберётся, просто счётчик не будет крутиться.

## Как написать статью

Статьи лежат в `src/content/articles/`, формат `.md` или `.mdx`.
`.mdx` нужен, если хочешь вставить квиз или редактор Python.
Имя файла = слаг в URL: `my-article.mdx` → `/article/my-article/`.

### Frontmatter

В начале файла обязательный блок с метаданными:

```yaml
---
title: 'Заголовок статьи'
description: 'Короткое описание для превью и SEO'
pubDate: 'May 17 2026'              # формат: Month Day Year
updatedDate: 'May 20 2026'          # необязательно
heroImage: '../../assets/cover.jpg' # необязательно, путь от файла статьи
category: 'Python'                  # одна из категорий ниже
author: 'Имя автора'                # необязательно, по умолчанию 'Николай'
---
```

Категории (живут в `src/consts.ts`):

- `Python`
- `Задания`
- `Экзамен`
- `Всякая всячина`

Всё это проверяется Zod-схемой в `src/content.config.ts`. Если налажал
с типами, сборка упадёт и подскажет, где именно.

### Ссылки

- **Наружу** - обычный Markdown: `[текст](https://example.com)`
- **На другие статьи** - компонент `<Link>` (только в `.mdx`):

```mdx
import { Link } from '../../components/Link.tsx'

Подробнее в статье <Link link='/article/lru-cache/' name='про @lru_cache' />.
```

### Компонент Quiz

Вопрос с вариантами ответов и пояснениями. Импортится в `.mdx`:

```mdx
import { Quiz } from '../../components/Quiz.tsx'
```

Если в вопросе есть код, удобнее писать его как children:

```mdx
<Quiz
  client:load
  options={['"null"', '"object"', '"undefined"', 'TypeError']}
  correct={1}
  explanations={{
    0: 'Не совсем - typeof возвращает не само значение, а его тип строкой.',
    1: 'Верно! Это известный баг JavaScript с первой версии языка.',
    2: 'Нет - undefined и null это разные значения с разными типами.',
    3: 'Нет, ошибки не будет - typeof работает с любым значением.'
  }}
>
  Что вернёт <code>typeof null</code>?
</Quiz>
```

Если вопрос простой и без форматирования, можно через проп:

```mdx
<Quiz
  client:load
  question="Что такое замыкание?"
  options={['Функция внутри функции', 'Функция с захваченной областью видимости', 'Глобальная переменная']}
  correct={1}
/>
```

`explanations` можно не передавать. Тогда после ответа будет только
«правильно/неправильно», без подробностей.

### Компонент PythonEditor

Редактор, который гоняет Python в браузере через Pyodide:

```mdx
import { PythonEditor } from '../../components/PythonEditor.tsx'

<PythonEditor
  client:only="react"
  timeoutMs={5000}
  initialCode={`# Задача 1 ЕГЭ: сумма цифр числа
n = 1234
result = 0
while n > 0:
    result += n % 10
    n //= 10
print(result)`}
/>
```

Пропсы:

- `initialCode` - что лежит в редакторе при загрузке
- `timeoutMs` - таймаут на выполнение в мс, по умолчанию 5000
- `client:only="react"` - обязательно, иначе Pyodide не подцепится (он только в браузере)
- `theme` - подсветка синтаксиса. По умолчанию `idle`, можно явно передать, тогда возможность смены подсветки на компоненте заблокируется.

### Математика

LaTeX через KaTeX, работает и в `.md`, и в `.mdx`:

- Инлайн: `$E = mc^2$`
- Блочно:

```latex
$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$
```

## Как прислать свою статью

Тема - что-то связанное с информатикой, comp science, ЕГЭ или Python. Делай PR, я буду рад.

### По шагам

1. **Форкни** репу кнопкой "Fork" на GitHub.

2. **Клонируй форк** и поставь зависимости:
   ```bash
   git clone https://github.com/<твой-username>/OpenKEGE-Articles.git
   cd OpenKEGE-Articles
   npm install
   ```

3. **Создай ветку** с понятным именем:
   ```bash
   git checkout -b article/название-статьи
   ```

4. **Положи статью** в `src/content/articles/`. Имя файла станет слагом URL,
   так что пиши латиницей в кебаб-кейсе: `binary-search-tricks.mdx`, а не
   `Бинарный поиск.mdx`.

5. **Если есть обложка**, клади её в `src/assets/` и пропиши путь
   во frontmatter (`heroImage: '../../assets/your-image.jpg'`).
   Постарайся ужать до ~500 КБ, например через [Squoosh](https://squoosh.app/).

6. **Проверь сборку**:
   ```bash
   npm run build
   ```
   Если без ошибок, frontmatter в порядке и Zod-схема довольна.

7. **Посмотри глазами**. Запусти `npm run dev` и убедись, что квизы кликаются,
   Python запускается, ссылки никуда не уезжают.

8. **Коммить и пушь:**
   ```bash
   git add .
   git commit -m "Add: статья про бинарный поиск"
   git push origin article/название-статьи
   ```

9. **Открой Pull Request** сюда. В описании пары предложений хватит -
   о чём статья и для кого.

### Что мне важно

Полезно по теме, написано по-русски без жести с опечатками, frontmatter
заполнен, имя файла не дублирует уже существующие. Плагиат не пройдёт -
либо пиши своими словами, либо ссылайся на источник. Без приколов не принимается.
