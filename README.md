# OpenKEGE Articles

Статьи по информатике для подготовки к ЕГЭ. Сайт построен на Astro и сосредоточен на чтении: помимо 
текста доступны интерактивные квизы и встроенный редактор Python для запуска кода прямо в браузере. 
И не только, компоненты дописываются по мере их надобности.

## Стек технологий

- **Astro 6** -- фреймворк для контентных сайтов
- **React 19** -- интерактивные островки (квизы, редактор кода)
- **MDX** -- Markdown с поддержкой JSX-компонентов
- **Tailwind CSS 4** -- стилизация
- **KaTeX** -- рендеринг математических формул
- **Pyodide** -- выполнение Python-кода в браузере
- **Supabase** -- счётчик лайков под статьями

## Запуск проекта локально

```bash
# 1. Клонировать репозиторий
git clone https://github.com/CompleWin/OpenKEGE-Articles.git
cd OpenKEGE-Articles

# 2. Установить зависимости
npm install

# 3. Создать .env файл (см. раздел ниже)
cp .env.example .env  # если есть пример, иначе создайте вручную

# 4. Запустить дев-сервер
npm run dev
```

Сайт будет доступен по адресу `http://localhost:4321`.

## Переменные окружения

В корне проекта должен лежать файл `.env` со следующими переменными (нужны для работы кнопки лайков):

```env
PUBLIC_SUPABASE_URL=https://your-project.supabase.co
PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

> Если ты только пишешь статьи и не трогаешь лайки, можно оставить эти переменные пустыми — сайт соберётся, но счётчик лайков работать не будет.

## Как написать статью

Статьи лежат в `src/content/articles/` и могут быть в формате `.md` или `.mdx`. 
Используй `.mdx`, если хочешь вставлять интерактивные компоненты (квизы, Python-редактор).
Имя файла становится слагом URL: `my-article.mdx` -> `/article/my-article/`.

### Frontmatter

В начале каждой статьи обязательно идёт блок с метаданными:

```yaml
---
title: 'Заголовок статьи'
description: 'Короткое описание для превью и SEO'
pubDate: 'May 17 2026'              # Формат: Month Day Year
updatedDate: 'May 20 2026'          # необязательно
heroImage: '../../assets/cover.jpg' # необязательно — путь относительно файла статьи
category: 'Python'                  # одна из категорий ниже
author: 'Имя автора'                # необязательно, по умолчанию 'Николай'
---
```

**Доступные категории** (см. `src/consts.ts`):

- `Python`
- `Задания`
- `Экзамен`
- `Всякая всячина`

> Схема валидируется автоматически через Zod в `src/content.config.ts` — если что-то заполнено неправильно, сборка упадёт с понятной ошибкой.

### Ссылки

- **Внешние ссылки** — обычный Markdown: `[текст](https://example.com)`
- **Ссылки на другие статьи** — компонент `<Link>` (только в `.mdx`):

```mdx
import { Link } from '../../components/Link.tsx'

Подробнее в статье <Link link='/article/lru-cache/' name='про @lru_cache' />.
```

### Компонент Quiz

Интерактивный вопрос с вариантами ответов и пояснениями. Импортируется в `.mdx`:

```mdx
import { Quiz } from '../../components/Quiz.tsx'
```

**Вариант 1 -- вопрос как children** (удобно, если в вопросе есть код):

```mdx
<Quiz
  client:load
  options={['"null"', '"object"', '"undefined"', 'TypeError']}
  correct={1}
  explanations={{
    0: 'Не совсем — typeof возвращает не само значение, а его тип строкой.',
    1: 'Верно! Это известный баг JavaScript с первой версии языка.',
    2: 'Нет — undefined и null это разные значения с разными типами.',
    3: 'Нет, ошибки не будет — typeof работает с любым значением.'
  }}
>
  Что вернёт <code>typeof null</code>?
</Quiz>
```

**Вариант 2 -- вопрос через проп** (для простых вопросов без форматирования):

```mdx
<Quiz
  client:load
  question="Что такое замыкание?"
  options={['Функция внутри функции', 'Функция с захваченной областью видимости', 'Глобальная переменная']}
  correct={1}
/>
```

Поле `explanations` необязательно, без него после ответа покажется только "правильно/неправильно".

### Компонент PythonEditor

Редактор Python с выполнением кода прямо в браузере через Pyodide:

```mdx
import { PythonEditor } from '../../components/PythonEditor.tsx'

<PythonEditor
  client:only="react"
  timeoutMs={5000}
  initialCode={`# Задача 1 ЕГЭ: сумма цифр числа
n = 1234
result = 0
while n > 0:
    result += n % 10
    n //= 10
print(result)`}
/>
```

**Пропсы:**

- `initialCode` -- начальный код в редакторе
- `timeoutMs` -- таймаут выполнения в миллисекундах (по умолчанию `5000`)
- `client:only="react"` -- обязательно, Pyodide работает только в браузере

### Математические формулы

В Markdown/MDX поддерживается LaTeX через KaTeX:

- **Инлайн:** `$E = mc^2$`
- **Блочные:**

```latex
$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$
```

## Контрибьютинг: как прислать свою статью

Проект открыт для контрибьюторов. Если хочешь поделиться своей статьей по информатике,
ЕГЭ, Python или смежным темам, то делай Pull Request.

### Пошаговая инструкция

1. **Сделай форк** репозитория через кнопку "Fork" на GitHub.

2. **Склонируй свой форк** и перейди в папку:
   ```bash
   git clone https://github.com/<твой-username>/OpenKEGE-Articles.git
   cd OpenKEGE-Articles
   npm install
   ```

3. **Создай новую ветку** с понятным названием:
   ```bash
   git checkout -b article/название-статьи
   ```

4. **Добавь статью** в `src/content/articles/`. Имя файла это слаг URL, поэтому используй 
5. кебаб-кейс латиницей: `binary-search-tricks.mdx`, а не `Бинарный поиск.mdx`.

5. **Если у статьи есть hero-картинка**, положи её в `src/assets/` и сошлись на неё 
6. в frontmatter (`heroImage: '../../assets/your-image.jpg'`). 
7. Желательно ужать картинку до разумного размера (<= 500 КБ) например, через [Squoosh](https://squoosh.app/).

6. **Проверь, что сайт собирается**:
   ```bash
   npm run build
   ```
   Если сборка прошла без ошибок, frontmatter корректен и Zod-схема довольна.

7. **Прогони статью локально** (`npm run dev`) и убедись, что всё отображается как надо: 
8. квизы кликаются, Python-код запускается, ссылки работают.

8. **Закоммить и запушь:**
   ```bash
   git add .
   git commit -m "Add: статья про бинарный поиск"
   git push origin article/название-статьи
   ```

9. **Открой Pull Request** в этот репозиторий. В описании PR кратко расскажи, 
10. о чём статья и для какой аудитории.

### Что я жду от статьи

- ✅ Полезный материал по теме информатики / ЕГЭ / программирования
- ✅ Грамотный русский язык
- ✅ Корректный frontmatter с выбранной категорией
- ✅ Уникальное имя файла, не пересекающееся с уже существующими статьями
- ❌ Без плагиата, пиши своими словами или ссылайся на источники
- ❌ Без приколов не принимается

### Что происходит дальше

PR будет ревьюиться мной, возможны правки и обсуждение. После одобрения статья мерджится в `main` 
и автоматически появляется на сайте при следующем деплое.

> 💡 Если ты не уверен, подходит ли тема, или хочешь обсудить идею до того как писать, открой
> [Issue](https://github.com/CompleWin/OpenKEGE-Articles/issues) или напиши мне в [telegram](https://t.me/nikolayivakhnenko).

### Авторство

Имя автора берётся из поля `author` во frontmatter и показывается под статьей. Если поле не заполнено,
по умолчанию подставляется мое, поэтому не забудь указать своё имя!
